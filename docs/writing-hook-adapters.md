# Writing Hook Adapters

Hook adapters integrate Linguo with AI coding tools. When a developer edits a file, the hook triggers extraction and sends the results to the AI tool, which generates translations automatically.

Linguo ships with a Claude Code adapter (`@gavai/linguo-hook-claude`). You can write adapters for Cursor, Windsurf, Copilot, or any tool that supports hooks or file watchers.

## HookAdapter Interface

```ts
import type { ExtractionResult, TranslationResult, HookConfig } from "@gavai/linguo";

/** Plugin interface for integrating with AI coding tool hook systems. */
interface HookAdapter {
  /** Name of the tool this adapter integrates with (e.g., "claude-code", "cursor") */
  name: string;

  /**
   * Generate hook config files for the target tool.
   * Called during `linguo init` to install hook configuration.
   * @returns Array of files to write (path + content)
   */
  install(): Promise<HookConfig[]>;

  /**
   * Format extraction results into a prompt/request for the AI tool.
   * @param extracted - Strings extracted from source files
   * @returns A formatted string (typically a prompt) to send to the AI tool
   */
  formatRequest(extracted: ExtractionResult[]): string;

  /**
   * Parse the AI tool's translation response.
   * @param raw - The raw response string from the AI tool
   * @returns Parsed translation results grouped by language
   */
  parseResponse(raw: string): TranslationResult[];
}
```

### Supporting Types

```ts
/** Hook configuration generated by a HookAdapter. */
interface HookConfig {
  /** Path to write the hook config file */
  path: string;
  /** Content of the hook config file */
  content: string;
}

/** A batch of translations for a single target language. */
interface TranslationResult {
  language: string;
  translations: Record<string, string>;
}
```

## How Hook-Driven Translation Works

1. The developer runs `linguo init`. The adapter's `install()` writes hook config files for the target tool.
2. The developer edits a source file. The tool's hook system triggers `linguo extract --format hook-output`, which outputs extraction results as JSON.
3. The adapter's `formatRequest()` converts the extraction JSON into a prompt the AI tool understands.
4. The AI tool generates translations and returns a response.
5. The adapter's `parseResponse()` extracts structured `TranslationResult[]` from the raw response.
6. Linguo writes the translations to locale files via the configured formatter.

## The install/formatRequest/parseResponse Contract

### `install()`

Returns an array of `HookConfig` objects. Each one describes a file to write:

- `path` -- Relative path from the project root (e.g., `.cursor/hooks/linguo.json`)
- `content` -- Full file content as a string

The hook config should tell the AI tool to run `linguo extract --format hook-output` when relevant files change.

### `formatRequest(extracted)`

Takes an `ExtractionResult[]` and returns a string prompt. The prompt should:

- List all strings that need translation
- Include context hints where available
- Specify the expected response format so the AI tool returns structured data

### `parseResponse(raw)`

Takes the raw string response from the AI tool and returns `TranslationResult[]`. This function must be resilient:

- Extract JSON from markdown code blocks or mixed text
- Handle partial responses gracefully
- Return an empty array if parsing fails (never throw)

## Example: Cursor Hook Adapter

Here is a complete adapter for a hypothetical Cursor hooks system.

### Package setup

```json
{
  "name": "@gavai/linguo-hook-cursor",
  "version": "0.1.0",
  "type": "module",
  "exports": {
    ".": {
      "types": "./dist/index.d.ts",
      "import": "./dist/index.js",
      "require": "./dist/index.cjs"
    }
  },
  "dependencies": {
    "@gavai/linguo": "workspace:*"
  }
}
```

### Implementation

```ts
// packages/hook-cursor/src/adapter.ts
import type {
  HookAdapter,
  ExtractionResult,
  TranslationResult,
  HookConfig,
} from "@gavai/linguo";

export function createCursorHookAdapter(): HookAdapter {
  return {
    name: "cursor",

    async install(): Promise<HookConfig[]> {
      const config = {
        hooks: {
          onFileSave: {
            pattern: "src/**/*.{ts,tsx,vue}",
            command: "linguo extract --format hook-output $FILE",
          },
        },
      };

      return [
        {
          path: ".cursor/hooks/linguo.json",
          content: JSON.stringify(config, null, 2),
        },
      ];
    },

    formatRequest(extracted: ExtractionResult[]): string {
      const entries = extracted.map((e) => ({
        key: e.key,
        text: e.text,
        context: e.context ?? null,
      }));

      return [
        "Translate the following strings into the target languages.",
        "Respond with JSON in this exact format:",
        "",
        '{"translations": [{"language": "<code>", "translations": {"<key>": "<translated>"}}]}',
        "",
        "Strings:",
        JSON.stringify(entries, null, 2),
      ].join("\n");
    },

    parseResponse(raw: string): TranslationResult[] {
      // Try to find JSON in the response (may be wrapped in markdown)
      const jsonMatch = raw.match(/\{[\s\S]*"translations"[\s\S]*\}/);
      if (!jsonMatch) return [];

      try {
        const parsed = JSON.parse(jsonMatch[0]) as {
          translations: TranslationResult[];
        };
        return parsed.translations;
      } catch {
        return [];
      }
    },
  };
}
```

### Public API

```ts
// packages/hook-cursor/src/index.ts
import { createCursorHookAdapter } from "./adapter.js";

export function cursorHookAdapter() {
  return createCursorHookAdapter();
}
```

### Registration

```ts
// linguo.config.ts
import { defineConfig } from "@gavai/linguo";
import { reactExtractor } from "@gavai/linguo-react";
import { jsonFormatter } from "@gavai/linguo-json";
import { cursorHookAdapter } from "@gavai/linguo-hook-cursor";

export default defineConfig({
  sourceLanguage: "en",
  targetLanguages: ["es", "fr"],
  include: ["src/**/*.tsx"],
  outDir: "locales",
  extractors: [reactExtractor()],
  formatters: [jsonFormatter()],
  hookAdapters: [cursorHookAdapter()],
});
```

## Testing

Test each method independently:

```ts
import { describe, expect, test } from "bun:test";
import { createCursorHookAdapter } from "../src/adapter.js";

describe("Cursor hook adapter", () => {
  const adapter = createCursorHookAdapter();

  test("install returns hook config", async () => {
    const configs = await adapter.install();

    expect(configs).toHaveLength(1);
    expect(configs[0]!.path).toBe(".cursor/hooks/linguo.json");
    expect(JSON.parse(configs[0]!.content)).toHaveProperty("hooks");
  });

  test("formatRequest includes all keys", () => {
    const extracted = [
      {
        text: "Hello",
        key: "hero.hello",
        location: { file: "Hero.tsx", line: 1, column: 0 },
      },
    ];

    const request = adapter.formatRequest(extracted);

    expect(request).toContain("hero.hello");
    expect(request).toContain("Hello");
    expect(request).toContain("translations");
  });

  test("parseResponse extracts translations from JSON", () => {
    const raw = JSON.stringify({
      translations: [
        { language: "es", translations: { "hero.hello": "Hola" } },
      ],
    });

    const results = adapter.parseResponse(raw);

    expect(results).toHaveLength(1);
    expect(results[0]!.language).toBe("es");
    expect(results[0]!.translations["hero.hello"]).toBe("Hola");
  });

  test("parseResponse handles invalid input", () => {
    expect(adapter.parseResponse("not json")).toEqual([]);
    expect(adapter.parseResponse("")).toEqual([]);
  });
});
```

## Tips

- Keep `parseResponse()` defensive. AI tools return unpredictable formatting.
- Use `--format hook-output` in your hook command to get JSON output from `linguo extract`.
- Test with real AI tool responses when possible -- capture a few examples and use them as test fixtures.
